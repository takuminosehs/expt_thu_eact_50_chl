# 2.目的・手法
グローバルのみでやったほうがローカルを入れて更新するよりも精度がいい結果になってしまったので、それを是正するために以下のことをする。
1. グローバルの更新 -> ローカルの更新 -> 両方同時の更新 という形で、50エポックずつ行う。ローカルの更新の時には、グローバルの重みは全く動かさない。
2. ローカルのテンソルの作り方を、「一つのファイルを時間で４等分し、それぞれでイベントが密集している部分を抜き出して重ね合わせる」というものにする。

# 3.結果・考察
Epoch 128 [STAGE-3 (All-Synch) | G-LR: 0.000131 | L-LR: 0.000131 | Gt-LR: 0.004373] -> Loss: 0.7064 | Train: 100.00% | ★Test: 75.74% | [G-Gate: 0.987 / L-Gate: 0.013]
で、一応両方使ったところが精度最大となったが、そもそもの精度が上がりきらず、/home/nose/ghq/github.com/takuminosehs/expt_thu_eact_50_chl/experiments/260709_3と比べた時に、2%強落ちていた。
これは、これまで100エポックで学習率を0にしていたところを50で0にするように設定していたため、動きを出せるエポックを限定してしまったのが原因と考えられる。

# 4.今後の方針
いったん、ローカルテンソルの抜出がうまく行われているかを確認する。
その後、ローカルテンソルの詳細部分を抜き出す手法を工夫し、精度の向上を図る。
