# 2.目的・手法
「携帯電話で電話をかける」が「敬礼をする」と間違えられていることから、細かい形状を見分けられる仕組みが必要であると考えた。
そこで、チャンネル注意（どのような特徴が重要か） と 空間注意（どこが重要か）を適応的に付与するモジュールであるCBAM（Convolutional Block Attention Module）を導入する。（ResNet-18を用いているもので探してしまったので、ほかにもいいのがあるかもしれない。）元となる論文は以下の通り。/home/nose/ghq/github.com/takuminosehs/expt_thu_eact_50_chl/references/Sanghyun_Woo_Convolutional_Block_Attention_ECCV_2018_paper.pdf
この方法と、HoloEv-Netを掛け合わせてどうなるか見る。
また、このモデルを使用して、「携帯電話で電話をかける」が「敬礼をする」を間違える件数が少なくなったのかを調べる。

# 3.結果・考察
Epoch 078/100 [LR: 0.000037] -> Loss: 0.7052 | Train Top-1: 100.00% (Top-5: 100.00%)| ★Test Top-1: 76.29% (Top-5: 94.49%)
これが最高精度となり、元となっているモデルの
Epoch 051/100 [LR: 0.000150] -> Loss: 0.7086 | Train Top-1: 100.00% (Top-5: 100.00%)| ★Test Top-1: 77.39% (Top-5: 93.75%)
という記録より1%近く下回りました。
「携帯電話で電話をかける」が「敬礼をする」を間違える件数は7件から5件になっていましたが、ほかの部分をまばらに間違えるようになり、どんなデータに対してもまばらに間違えるタイプのモデルになったようです。

# 4.今後の方針
cbam1 と cbam2 を完全に撤去し、大まかな動作の構造が完成する cbam3 と cbam4（あるいは cbam4 のみ）に絞ります。これにより、低次のノイズを増幅させるリスクを排除しつつ、目的の「手元の物体認識」だけに注意を機能させる方針にします。
