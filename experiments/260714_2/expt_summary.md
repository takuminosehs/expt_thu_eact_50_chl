# 2.目的・手法
これまでの決定方法では、グローバルの予測が高すぎて、ローカルとゲートを変数としてもほとんど動かないという現象が起こっていると予測した。（根拠としては、 [G-Gate: 1.000 / L-Gate: 0.000]の状態が常に続いていたため。）ローカルが何も予測できていない状態のモノを使っても、それを重視する方向には動かない。よって、以下の方法での更新を行う。
1. グローバル単体で重みを決定
2. ローカル単体で重みを決定
3. ゲート単体で重みを決定
4. すべてを動かして重みを決定
それぞれのエポック数、および学習率の初期値はそれぞれ定められるようにする（例えば、ステップ1のグローバルの学習率の初期値、ステップ4のグローバルの学習率の初期値というように、個別指定が可能なようにする。）。
